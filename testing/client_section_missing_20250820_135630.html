#### Job Title:
Python/AI Developer to Build Secure PDF Data Extraction System and Private Lender Database

#### Job Description:
I'm a commercial real estate mortgage broker looking to hire an experienced Python developer with AI expertise to automate the creation of a secure, private database of lenders. I have over 300 PDF "tear sheets" containing lender information (e.g., names, loan types, rates, contacts), and I need a system that:

- Ingests and parses PDFs to extract key data (handling text, tables, and possibly OCR for scanned docs).
- Uses AI to structure the extracted data, generate summaries, and create a searchable summary table (e.g., in CSV/Excel format or a dynamic view).
- Stores everything in a private database (e.g., vector DB for semantic search) that only I can access—fully local or on a secure cloud under my control.
- Allows easy updates with new PDFs, appending or updating records without duplicates.
- Provides a simple interface (e.g., web app or dashboard) for searching the data naturally (e.g., "Find lenders for multifamily loans under $5M") and generating reports/tables.

The system should be secure, private, and scalable. No public cloud sharing—everything stays on my machine or a private setup. I'll provide sample PDFs to start.

**Key Deliverables:**
- Fully working script or app for PDF ingestion, extraction, and database population.
- AI-powered summarization and table generation.
- Searchable interface with update functionality.
- Documentation on how to use and maintain it.
- Test with a batch of my PDFs to ensure accuracy.

**Tech Stack Preferences (based on recommendations):**
- Python for core scripting.
- Libraries like pdfplumber or PyMuPDF for PDF parsing, LlamaIndex or LangChain for AI orchestration, Chroma or FAISS for vector database, Streamlit for a simple UI.
- OpenAI or open-source LLMs (e.g., Hugging Face models) for data extraction/summarization.
- Ensure it's runnable locally for privacy.

If you have experience with similar projects (e.g., PDF data extraction for databases), please share examples from your portfolio. I'm not tech-savvy, so I need someone who can handle the full build and explain it simply.

I've attached a couple sample pdf's to serve as an example of the files we'll be extracting data from.

#### Skills Required:
- Python
- AI/ML (especially NLP and data extraction)
- PDF Parsing and OCR
- Database Management (SQL/NoSQL, Vector DBs)
- Web Development (e.g., Streamlit or similar for dashboards)
- Data Processing (Pandas, NumPy)
- API Integration (e.g., for LLMs)

#### Project Type:
Fixed-Price (preferred) or Hourly

#### Budget:
$500 - $1,000 (depending on scope; open to proposals based on your estimate)

#### Timeline:
1-2 weeks for initial build, with possible 1-week extension for testing/updates.
